# Interview Talking Points

## 30-second explanation
I created a small vulnerability prioritization project based on the tools mentioned for the role: Tenable and Wiz. The goal was to show how I would take scanner findings, add cloud exposure context, and turn that into a ranked remediation report.

## Why I built it
For vulnerability management, the biggest issue is usually not the lack of findings. It is having too many findings and needing to know what truly creates business risk. This project shows how I would prioritize vulnerabilities based on exploitability, exposure, asset criticality, and cloud attack path context.

## How it connects to Colas
Colas operates in a large, distributed environment, so prioritization matters. A global organization may have many subsidiaries, systems, cloud workloads, and user environments. A risk-based process helps remediation teams focus on the highest-impact issues first.

## Simple answer if asked why this stands out
I wanted to show initiative before the interview. Since this role is heavily focused on vulnerability analysis, I built something practical that reflects how I think: validate the finding, understand the business impact, prioritize the risk, coordinate remediation, and report clearly.
